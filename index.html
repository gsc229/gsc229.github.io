For image management, I implemented the Cloudinary image management platform. The Cloudinary API provides users a wide range of options to customize their profiles from multiple image upload sources such as Facebook, Instagram, Dropbox, or their local drive. Users perform transformations on image shapes, borders, hues, saturations and more – all on the fly!

I used React with hooks to build the vendor profile section for the front end. React easily allowed me to manage the state of my application. For example, you can line-item edit the vendor's information by clicking directly on it and instantly save changes to the database.

Market Ave. Api

Market Ave. uses Node, Express, Mongo, and Mongoose on the backend. The API is very robust and there's a ton of things you can do with it:

  • search vendors in the vicinity of a zipcode
  • filter vendors by their selling category
  • filter food categories
  • easily get the data of multiple vendors, their products, and images all in one simple request to the API
  • trim down data to only get back the information you need
  • set limits on the amount of data per page for paginiation
    • ...and so much more.

Link to my API documentation for this project:
Market Ave API documentation

You can try it out by making some queries using this base URL: https://quickstlabs.herokuapp.com/api/v1.0
If you query by zip code, you can use 18641 as this was our test zip code in development and there are plenty of vendors around. (Note: if the server has been inactive for a while, your first query may take a few seconds)
